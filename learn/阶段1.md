## 2. 整合后的 Ghidra 脚本

文件：`integrated/scripts/TLShunterAnalyzer.java`

### 已实现的阶段 1 目标

脚本已按单次执行顺序完成以下分析入口：
1. `HKDF`：搜索 `"s hs traffic"`
2. `SSL_LOG_SECRET`：搜索 `"EXPORTER_SECRET"` / `"CLIENT_RANDOM"`
3. `PRF`：搜索 `"master secret"`
4. `KEY_EXPANSION`：搜索 `"key expansion"`

### 共享基础设施

已抽取出统一的基础能力：
- Ghidra 已识别字符串搜索
- 只读数据段搜索（`.rodata` / `.rdata` / `__cstring` / `__const`）
- XREF → 函数收集
- 数据引用追踪
- 从函数入口提取前缀指纹
- 基于 `imageBase` 计算 RVA

### 当前统一输出格式

脚本会输出统一的机器可解析行：

```text
[RESULT] type=HKDF function=FUN_xxx rva=0xXXXXXXXX fingerprint=55 48 89 E5 ...
[RESULT] type=SSL_LOG_SECRET function=FUN_xxx rva=0xXXXXXXXX fingerprint=55 48 89 E5 ...
[RESULT] type=PRF function=FUN_xxx rva=0xXXXXXXXX fingerprint=55 48 89 E5 ...
[RESULT] type=KEY_EXPANSION function=FUN_xxx rva=0xXXXXXXXX fingerprint=55 48 89 E5 ...
```

### 当前实现说明

这是阶段 1 的“基础整合版”：
- HKDF / PRF / key_expansion 先统一走同一类字符串/XREF 逻辑
- `ssl_log_secret` 额外实现了“找到字符串引用函数后，继续寻找其后续调用目标函数”的逻辑
- PRF 目前**仍未修复** Chrome/BoringSSL 上的 `.rodata` 子串问题，这符合阶段 1 预期，后续在阶段 2 修复

## 3. Docker 环境

文件：`integrated/Dockerfile`

### 已实现内容

- 使用统一的 Ghidra 版本：`12.0.3`
- 复用 `TLSKeyHunter/MinimalAnalysisOption.java`
- 复用 `TLSKeyHunter/custom_log4j.xml`
- 容器启动后执行：
  - `analyzeHeadless`
  - `TLShunterAnalyzer.java`
- 输入目录：`/usr/local/src/binaries`
- 输出：
  - 控制台中的 `[RESULT]` 行
  - `/host_output/analysis.log`

文件：`integrated/ghidra_analysis.sh`

该脚本负责：
- 从 `/usr/local/src/binaries/` 中挑选第一个支持的 ELF / Mach-O / PE32 文件
- 调用 Ghidra headless 运行整合分析脚本
- 通过 `tee` 同时输出到控制台与 `/host_output/analysis.log`

## 4. Python 入口脚本

文件：`integrated/run.py`

### 已实现内容

`run.py` 已支持：
1. 自动构建 Docker 镜像
2. 挂载输入二进制目录并运行容器
3. 解析控制台输出中的 `[RESULT]` 行
4. 组装基础 JSON 输出
5. 将结果写入用户指定的输出文件

### 当前命令行用法

```bash
python3 TLShunter/integrated/run.py \
  --binary artifacts/chrome/143.0.7499.169/chrome \
  --output results/143.0.7499.169.json
```

### 当前 JSON 结构

当前阶段 1 输出的是基础兼容结构：

```json
{
  "meta": {
    "binary": "chrome",
    "analysis_tool": "TLShunter phase1",
    "analysis_date": "..."
  },
  "hook_points": {
    "hkdf": { ... },
    "ssl_log_secret": { ... },
    "prf": { ... },
    "key_expansion": { ... }
  }
}
```

每个 hook point 当前包含：
- `function`
- `rva`
- `fingerprint`
- `fingerprint_len`
- `role`

完整对齐 `hooks/chrome_*.json` 的最终字段，将在后续阶段补齐。
