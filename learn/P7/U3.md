# U3-lite：统一 CLI 低风险整理状态

## 1. 目标

本阶段只做统一 CLI 的低风险整理，不改 runtime hook 核心逻辑，不删除旧入口，不删除 `tls_capture/hooks/*.json`。

目标是让：

- `pip install -e .` 后的 `tshunter` 命令可以作为主入口使用；
- 各子命令的 `--help` 都可稳定工作；
- `batch` 子命令不再静默返回成功，而是明确提示当前未实现；
- 增补 CLI smoke tests，保证后续重构不会把命令入口弄坏。

---

## 2. 本阶段明确不做的事

为了保持低风险，本阶段刻意**不做**以下事情：

- 不修改 `tshunter/config_loader.py` 核心逻辑；
- 不修改 runtime hook 核心逻辑；
- 不删除 `tls_capture/tls_capture.py`；
- 不删除 `tls_capture/hooks/*.json`；
- 不删除兼容 wrapper；
- 不把 U2/U3 变成新的结构性重构。

也就是说，U3-lite 的重点是：

**统一命令入口与测试，不碰运行时主链路。**
